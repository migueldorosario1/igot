# 🤝 Contribuindo para o igot

Obrigado pelo interesse! O projeto está começando, então neste momento o jeito
mais útil de contribuir é debater ideias, reportar problemas e ajudar a definir
o rumo técnico.

## 💬 Antes de escrever código

1. Leia o [**README**](./README.md) e o [**ROADMAP**](./ROADMAP.md) — garanta
   que sua ideia está alinhada com o espírito do projeto.
2. Abra uma **issue** descrevendo o que você quer fazer (antes de um PR grande).
3. Para bugs pequenos ou typos, pode mandar PR direto.

## 🛠️ Configurando o ambiente

```bash
git clone git@github.com:migueldorosario1/igot.git
cd igot
# (a definir: npm install / pnpm install quando houver código)
```

## 📋 Convenções

- **Mensagens de commit** em português ou inglês, no padrão _conventional commits_:
  `feat:`, `fix:`, `docs:`, `refactor:`, `chore:`
- **Branches:** `feat/...`, `fix/...`, `docs/...`
- **Código:** TypeScript com tipagem estrita (quando o código existir).

## 🧭 Princípios do projeto

Ao contribuir, mantenha estes princípios:

1. **Valor primeiro** — cada mudança entrega algo usável.
2. **Local-first** — livro e anotações pertencem ao leitor; a IA é um auxílio.
3. **Sem invenção** — respostas da IA sempre com grounding no texto.
4. **Um passo de cada vez** — simples antes de complexo.

## 🐛 Reportando problemas

Abra uma issue com:
- O que você esperava que acontecesse
- O que aconteceu de fato
- Passos pra reproduzir
- Ambiente (navegador, OS)
